# dot-claude

Git-managed Claude Code user configuration (`~/.claude`).

## What's Tracked

- `settings.json` - Main settings (hooks, MCP servers, plugins)
- `agents/` - Custom agents
- `commands/` - Custom slash commands
- `skills/` - Custom skills
- `output-styles/` - Output style configurations
- `plugins/installed_plugins.json` - Installed plugin list
- `plugins/known_marketplaces.json` - Marketplace config

## What's Ignored

- Session data, caches, logs
- Machine-specific files (`config.json`, `settings.local.json`)
- Browser data, telemetry, debug logs

## Setup

1. Clone this repo:
   ```bash
   git clone https://github.com/TakumaKira/dot-claude.git ~/Git/dot-claude
   ```

2. Backup existing config:
   ```bash
   mv ~/.claude ~/.claude.backup
   ```

3. Create symlink:
   ```bash
   ln -s ~/Git/dot-claude/.claude ~/.claude
   ```

4. Verify Claude Code works by running `claude` in your terminal.

## Notes

- `config.json` is auto-generated by Claude Code for API key approvals and should not be committed
- `settings.local.json` is for machine-specific settings that override `settings.json`
